# File Tagger

یک ابزار سبک، ساده و Local-First برای ثبت، دسته‌بندی و جست‌وجوی فایل‌های مختلف.

با File Tagger می‌توان برای فایل‌های موجود در بخش‌های مختلف سیستم، اطلاعات تکمیلی مانند عنوان، توضیحات، تاریخ، تگ، موضوع و افراد مرتبط ثبت کرد؛ بدون نیاز به دیتابیس، نصب نرم‌افزار یا راه‌اندازی سرور.

## کاربرد

این ابزار محدود به فایل‌های جلسات یا ویدئو نیست و می‌توان برای انواع فایل‌ها از آن استفاده کرد، از جمله:

* مستندات
* فایل‌های پروژه
* قراردادها
* فایل‌های صوتی و تصویری
* PDF
* Word
* Excel
* فایل‌های آموزشی
* گزارش‌ها
* فایل‌های پراکنده روی سیستم

## قابلیت‌ها

* ثبت فایل از هر بخش سیستم
* ثبت عنوان دلخواه برای فایل
* ثبت توضیحات
* ثبت تاریخ شمسی
* افزودن تگ
* افزودن و حذف تگ مستقیماً از روی کارت فایل
* جست‌وجو در عنوان، توضیحات، مسیر، تگ‌ها و موضوع
* فیلتر بر اساس نوع فایل
* ثبت دسته‌بندی یا موضوع
* ثبت افراد مرتبط
* نمایش نام، پسوند و حجم فایل
* کپی مسیر فایل
* ویرایش و حذف اطلاعات
* ذخیره اطلاعات در فایل JSON
* دریافت نسخه پشتیبان
* رابط فارسی و RTL
* بدون نیاز به Backend
* بدون نیاز به دیتابیس
* بدون نیاز به نصب

## نحوه استفاده

1. فایل `MeetIndexer.html` را با مرورگر Chrome یا Edge باز کنید.
2. روی دکمه «انتخاب فایل اطلاعات» کلیک کنید.
3. فایل `meetings.json` را انتخاب کنید.
4. روی «افزودن فایل» کلیک کنید.
5. فایل موردنظر را انتخاب کنید.
6. عنوان، توضیحات، تاریخ، تگ‌ها و سایر اطلاعات را وارد کنید.
7. اطلاعات را ذخیره کنید.

پس از ذخیره، اطلاعات مستقیماً در فایل JSON ثبت می‌شوند.

## انتخاب فایل اطلاعات

مرورگر به دلایل امنیتی اجازه نمی‌دهد فایل HTML بدون تأیید کاربر به فایل‌های سیستم دسترسی داشته باشد.

به همین دلیل، بعد از هر بار باز کردن یا Refresh صفحه باید دوباره فایل JSON انتخاب شود.

این رفتار بخشی از محدودیت امنیتی مرورگر است و به معنی حذف اطلاعات نیست.

## محل ذخیره اطلاعات

اطلاعات برنامه داخل LocalStorage ذخیره نمی‌شوند.

تمام اطلاعات داخل فایل JSON نگهداری می‌شوند:

```text
meetings.json
```

ساختار کلی فایل:

```json
{
  "version": 1,
  "updatedAt": "2026-06-17T10:30:00.000Z",
  "meetings": [
    {
      "id": "uuid",
      "title": "عنوان فایل",
      "fileName": "example.pdf",
      "filePath": "D:\\Documents\\example.pdf",
      "fileExtension": ".pdf",
      "fileSize": 1250000,
      "fileLastModified": "",
      "meetingDate": "1405/03/27",
      "description": "توضیحات فایل",
      "tags": [
        "مستندات",
        "پروژه"
      ],
      "project": "موضوع یا دسته‌بندی",
      "participants": [
        "نام فرد مرتبط"
      ],
      "createdAt": "",
      "updatedAt": ""
    }
  ]
}
```

در نسخه فعلی، برای حفظ سازگاری با نسخه اولیه، برخی نام‌های فیلد مانند `meetings` و `meetingDate` هنوز باقی مانده‌اند و در نسخه‌های بعدی عمومی‌تر خواهند شد.

## تاریخ شمسی

تاریخ فایل به‌صورت شمسی انتخاب و ذخیره می‌شود.

فرمت ذخیره تاریخ:

```text
1405/03/27
```

تاریخ به‌صورت رشته در JSON ثبت می‌شود تا وابستگی به منطقه زمانی یا تنظیمات ویندوز نداشته باشد.

## مدیریت سریع تگ‌ها

تگ‌ها علاوه بر فرم ویرایش، مستقیماً از روی کارت هر فایل قابل مدیریت هستند.

امکانات:

* افزودن تگ جدید
* حذف تگ
* جلوگیری از ثبت تگ تکراری
* ذخیره فوری در JSON
* جست‌وجو بر اساس تگ

## جست‌وجو

جست‌وجو در بخش‌های زیر انجام می‌شود:

* عنوان
* توضیحات
* نام فایل
* مسیر فایل
* تگ‌ها
* دسته‌بندی یا موضوع
* افراد مرتبط

## مرورگرهای پشتیبانی‌شده

برای استفاده کامل از قابلیت خواندن و نوشتن فایل JSON، مرورگرهای زیر پیشنهاد می‌شوند:

* Google Chrome
* Microsoft Edge

مرورگر باید از File System Access API پشتیبانی کند.

## محدودیت مسیر فایل

مرورگرهای وب به دلایل امنیتی معمولاً مسیر کامل واقعی فایل انتخاب‌شده را در اختیار JavaScript قرار نمی‌دهند.

در نسخه فعلی:

* نام فایل دریافت می‌شود
* پسوند دریافت می‌شود
* حجم دریافت می‌شود
* تاریخ آخرین تغییر دریافت می‌شود
* مسیر کامل فایل را می‌توان به‌صورت دستی وارد کرد

برای دریافت خودکار مسیر کامل فایل، در آینده می‌توان یک برنامه بسیار سبک ویندوزی به پروژه اضافه کرد.

## حریم خصوصی

File Tagger یک ابزار Local-First است.

* اطلاعات به سرور ارسال نمی‌شوند
* پروژه Backend ندارد
* فایل‌ها آپلود نمی‌شوند
* اطلاعات داخل سیستم کاربر باقی می‌مانند
* داده‌ها فقط در فایل JSON انتخاب‌شده ذخیره می‌شوند

## پشتیبان‌گیری

پیشنهاد می‌شود فایل JSON به‌صورت دوره‌ای Backup گرفته شود.

نمونه نام فایل Backup:

```text
file-tagger-backup-1405-03-27.json
```

همچنین می‌توان فایل JSON را در یکی از محل‌های زیر نگهداری کرد:

* درایو غیرسیستمی
* پوشه اشتراکی شبکه
* OneDrive
* Google Drive
* فضای Backup سازمانی

## نکات استفاده تیمی

در صورت استفاده چند نفر از یک فایل JSON مشترک، هم‌زمانی در ویرایش می‌تواند باعث تداخل شود.

نسخه فعلی برای موارد زیر مناسب‌تر است:

* استفاده شخصی
* استفاده تیمی محدود
* استفاده نوبتی از فایل JSON
* نگهداری JSON در پوشه اشتراکی

برای استفاده چندکاربره هم‌زمان، در آینده نیاز به Backend یا دیتابیس سبک خواهد بود.

## برنامه توسعه

موارد پیشنهادی برای نسخه‌های بعدی:

* تغییر نام ساختار داده از `meetings` به `items`
* تغییر `meetingDate` به `itemDate`
* تغییر نام فایل HTML به `FileTagger.html`
* تغییر نام فایل JSON به `file-tagger-data.json`
* باز کردن مستقیم فایل
* باز کردن محل فایل در Windows Explorer
* دریافت خودکار مسیر کامل فایل
* تشخیص فایل‌های حذف‌شده یا جابه‌جا‌شده
* پیشنهاد خودکار تگ
* استخراج متن PDF و Word
* جست‌وجو داخل محتوای فایل
* نسخه Desktop سبک
* پشتیبانی بهتر از استفاده تیمی
* ذخیره‌سازی روی SQLite
* Drag & Drop
* نمایش Thumbnail فایل‌ها
* تعریف چند فایل JSON
* Import و Export پیشرفته
